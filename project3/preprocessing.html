  • Preprocessing

    Preprocessing can be implemented client-side, typically in a browser. It usually comes in a javascript file in order for the preprocessing to function. In terms of server side preprocessing, there is typically a package that needs to be installed on the server in order for files to be recognized and processed into CSS.

    SASS Mechanisms

  • Mixins

    Mixins allow for the definition of multiple styles that can be dropped in anywhere into a single style rule. They cannot include top-level statements.

  • Variables

    SASS variables are assigned a name beginning with $, which can then be refered to anywhere a value would normally appear.

  • Functions

    Functions are used to define operations using SassScript values. They can be reused throughout the stylesheet and provide a large degree of readability.

  • Operators

    In addition to standard mathematical ones like + and *, Sass supports additonal operators such as == and !=, to check if values are identical, +, -, and / for concatenating strings, and others.

  • Nested Variables

    Nested variables allow for precisely what the title describes, which can be implemented to a great degree of usefulness in order to write more concise and readable code.
